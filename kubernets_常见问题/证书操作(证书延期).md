### 1.2.1 使用延长证书过期的方法解决k8s证书过期的问题（短期过期）

```sh
# 下载更新脚本：update-kubeadm-cert.sh
wget  https://github.com/yuyicai/update-kube-cert/blob/master/update-kubeadm-cert.sh

# 上传到所有节点，并给于执行权限
chmod +x update-kubeadm-cert.sh

# 执行脚本更新证书，  （如果集群有多个主节点，则在每个主节点上执行）
## 使用docker作为cri运行时：     默认值为 runtime
./update-kubeadm-cert.sh all --cri docker
## 使用containerd作为cri运行时： 默认值为 runtime
./update-kubeadm-cert.sh all --cri containerd

# 手动重启控制平面 Pods（必要）
运行该命令后，您应该重新启动控制平面 Pods。这是必需的，因为当前并非所有组件和证书都支持动态证书重新加载。静态 Pod 由本地 kubelet 管理，而不是由 API 服务器管理，因此 kubectl 不能用于删除和重启它们。要重启静态 Pod，你可以暂时从 /etc/kubernetes/manifests/ 中删除其清单文件并等待 20 秒（参见 KubeletConfiguration 结构中的 fileCheckFrequency 值。如果 Pod 不再位于清单目录中，kubelet 将终止它。然后，您可以将文件移回，在另一个 fileCheckFrequency 周期后，kubelet 将重新创建 Pod，并且组件的证书更新可以完成
```

### 1.2.2 使用containerd容器运行时更新证书 (短期过期)

```sh
# 1.备份原来证书
cp -r /etc/kubernetes /etc/kubernetes_bak
# 2.重新生成证书
kubeadm certs renew all
# 3.覆盖$HOME/.kube/config⽂件
cp -rp $HOME/.kube/config $HOME/.kube/config.bak
cp -i /etc/kubernetes/admin.conf $HOME/.kube/config

# 4. 重启kubelet，自动重建所有控制平面Pod（10秒生效）
systemctl daemon-reload
systemctl restart kubelet
# 4.1 等待30秒，让组件完全启动
sleep 30

### 1.2.3 集群证书长期过期,只重启kubelet不成功，使用下面方法 （长期过期）
# 4.移除kubelet.conf⽂件（如果重启不生效操作此步骤）
rm -rf /etc/kubernetes/kubelet.conf && rm /var/lib/kubelet/pki/* -rf

# 5.⽣成kubeconfig⽂件
kubeadm init --kubernetes-version=v1.28.0 phase kubeconfig kubelet --apiserver-advertise-address <xx.xx.xx.xx apiserver IP地址> --apiserver-bind-port 6443

## 注意：
a. 集群版本号
b. vip 或 apiserver地址和端⼝号

# 6. 重启kubelet、kube-apiserver、kube-controller-manager、kube-scheduler、calico-kube-controller、kube-proxy、⽹络插件
# 7.验证新证书⽂件和过期时间是否延⻓
openssl x509 -in /etc/kubernetes/pki/apiserver.crt -noout -text | grep Not
```

#### Worker节点操作指南（长期过期）

```sh
Worker节点操作指南
# 1. 『master节点操作』创建⽤于存放证书配置的⽬录
mkdir -p /home/certs
# 2.『Master节点操作，再scp⾄Worker节点』⽣成node节点kubelet相关⽂件
## ⽣成kubelet相关配置⽂件
kubeadm init --kubernetes-version=v1.27.0 phase kubeconfig kubelet --node-name  【hostname，Node节点主机名，可hostname查看】 --kubeconfig-dir /home/certs --apiserver-advertise-address <xx.xx.xx.xx apiserver IP地址> --apiserver-bind-port 6443

# 3. 删除Worker节点kubelet.conf⽂件
rm -rf /etc/kubernetes/kubelet.conf && rm -rf /var/lib/kubelet/pki/*

# 4. 拷⻉ kubelet.conf ⽂件到work 节点
scp /home/certs/kubelet.conf root@xx.xx.xx.xx:/etc/kubernetes/

# 5.重启kubelet
systemctl restart kubelet

# 6.验证新证书⽂件和过期时间是否延⻓
openssl x509 -in /etc/kubernetes/pki/apiserver.crt -noout -text | grep Not

###检查集群状态
kubectl get pod -n kube-system
kubectl get nodes
```
