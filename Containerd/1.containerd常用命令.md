containerd 核心命令速查手册（一页版·日常运维必备）
说明：优先用 crictl（K8s 场景首选，用法贴近 docker），ctr 需加 -n k8s.io 查看 K8s 相关资源，核心命令标红加粗，直接复制可用。
一、服务管理（核心，必记）
- 查看 containerd 状态：systemctl status containerd
- 重启 containerd 服务：systemctl restart containerd
- 设置开机自启：systemctl enable containerd
- 关闭开机自启：systemctl disable containerd
- 实时查看日志：journalctl -u containerd -f
二、crictl 命令（K8s 运维首选，核心高频）
1. 基础配置（首次使用必做）
cat > /etc/crictl.yaml << EOF
runtime-endpoint: unix:///run/containerd/containerd.sock
image-endpoint: unix:///run/containerd/containerd.sock
timeout: 10
debug: false
EOF
2. Pod 管理（核心）
- 查看所有 Pod（沙箱）：crictl pods
- 停止 Pod：crictl stopp PodID
- 删除 Pod：crictl rmp PodID
3. 容器管理（核心）
- 查看运行中容器：crictl ps
- 查看所有容器（含停止）：crictl ps -a
- 进入容器（高频）：crictl exec -it 容器ID /bin/bash
- 查看容器详情：crictl inspect 容器ID
- 查看容器日志：crictl logs 容器ID
- 实时查看日志（高频）：crictl logs -f 容器ID
4. 镜像管理（核心）
- 查看本地所有镜像：crictl images
- 拉取镜像：crictl pull 镜像名:tag（例：crictl pull nginx:latest）
- 删除镜像：crictl rmi 镜像ID/镜像名
- 查看镜像详情：crictl inspecti 镜像ID
三、ctr 命令（原生客户端，补充使用）
1. 命名空间（核心，必加参数）
- 查看所有命名空间：ctr namespaces ls
- 指定 K8s 命名空间操作（必加 -n k8s.io）：所有 ctr 命令后加 -n k8s.io
2. 核心操作（常用）
- 查看 K8s 镜像：ctr -n k8s.io images ls
- 查看 K8s 容器：ctr -n k8s.io containers ls
- 查看容器运行任务：ctr -n k8s.io tasks ls
- 进入 K8s 容器：ctr -n k8s.io tasks exec --exec-id test -t 容器ID /bin/bash
- 拉取镜像：ctr images pull 镜像名:tag
- 删除镜像：ctr images rm 镜像名:tag
- 清理无用镜像：ctr images prune
四、高频快捷命令（日常最常用，背会效率翻倍）
- 查找某个组件容器（例：etcd）：crictl ps | grep etcd
- 快速进入容器：crictl exec -it 容器ID /bin/bash
- 实时查看容器日志：crictl logs -f 容器ID
- 查看 K8s 所有容器：ctr -n k8s.io containers ls
- 重启 containerd：systemctl restart containerd
五、备注
1. 所有命令可直接复制到服务器终端执行，无需修改（除了容器ID、PodID需替换）；
2. 生产环境优先用 crictl，避免用 ctr 误操作非 K8s 资源；
3. 忘记命令时，打开此手册直接查找，核心命令已标红加粗，重点突出。
